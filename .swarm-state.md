## Key Decisions Made

### 1. Ready Signal Strategy
**Decision**: Use existing `window.__sceneReady` flag already present in src/core/main.ts:244  
**Rationale**: Saves from adding redundant code; flag is already set after animation loop starts  
**Alternative rejected**: Adding a new flag (redundant, src/ modification)

### 2. WASM Test Bounds
**Decision**: Derived from `assembly/constants.ts`:
- X/Z: [-128, 128] (grid: 16 cols × 16 units, origin -128)
- Y: [-100, 500] (particles can fall below terrain, reach into sky)

**Rationale**: Prevents silent physics corruption; any particle out-of-bounds is caught  
**Data source**: Verified against GRID_COLS, GRID_ROWS, GRID_CELL_SIZE, GRID_ORIGIN_X/Z constants

### 3. Server Detection in Smoke Test
**Decision**: Implemented port 4173 check; reuse existing server if running  
**Rationale**: CI/local dev may have server already running (e.g., `npm run preview` in another terminal); prevents port conflicts  
**Fallback**: Auto-starts if no server found

### 4. WASM Imports (env.abort, env.seed)
**Decision**: Provided minimal env object to AssemblyScript instantiation  
**Rationale**: WASM glue code (candy_physics.js) declares imports but doesn't export them; direct WASM instantiation requires manual import provision  
**Impact**: Test is self-contained, no external dependencies

## Known Issues and Warnings

### CloudBatcher Capacity Warnings (Expected)
```
WARNING: [CloudBatcher] Max capacity reached (400 / 400)
```
These are **expected and normal**. They indicate the cloud system reached its maximum allocation during procedural generation. The test passes despite these warnings.

### Audio Worklet Loading (Expected)
The test may log:
```
AudioSystem: Attempted to load Worklet from: http://localhost:4173/js/audio-processor.js
```
This is expected. The audio system has fallbacks if the worklet fails to load.

Subsequent runs are faster once caches warm up.
