# Swarm State — Issue #1265: Player Ground Level, Eye Height & Object Alignment

> Started: 2026-06-30
> Status: **SLICES 1–4 COMPLETE** — awaiting final verification on hardware with stable WebGPU.

## What We’re Solving

Unify and harden ground-height computation so the first-person camera stays smooth and batched instances (trees, mushrooms, roots/rocks) sit flush on the ground. This is the foundation for #1266 (walkable cloud platforms).

## Current Findings

- ✅ **Ground query centralized**: `src/systems/ground-system.ts` is now the single source for terrain + lake/island + platform overrides + cached height queries.
- ✅ **Lake constants unified**: `LAKE_BOUNDS`, `LAKE_BOTTOM`, `LAKE_ISLAND`, `isInLakeBasin`, `isOnLakeIsland` live in GroundSystem and are re-exported for compatibility.
- ✅ **Consumers routed**: `physics.core.ts`, `physics-core.ts`, `physics-states.ts`, `physics-updates.ts`, `camera-modes.ts`, `game-loop.ts`, `glitch-grenade.ts`, `rainbow-blaster.ts`, `main.ts`, `ground-heightmap.ts`, and `generation-utils.ts` all read from GroundSystem.
- ✅ **Eye height data-driven**: `CONFIG.player.eyeHeight` / `spawnEyeHeightY` replace hard-coded `1.8` and `5`; `CONFIG.ground.followLerpSpeed` / `followMaxStep` drive camera vertical smoothing.
- ✅ **C++ Y post-processed**: after `updatePhysicsCPP`, player Y is reconciled against the authoritative GroundSystem height.
- ✅ **Batcher placement verified**: world generation already calls `getUnifiedGroundHeight()` (delegated to GroundSystem); no manual `y + 0.1` hacks remain for trees/mushrooms.
- ✅ **Debug visualizer added**: `src/debug/ground-debug.ts` with `?debugHeights=1` / `?debugPlayer=1` overlays.
- No `RockBatcher` currently exists in the repo.

## Completed Work

1. ✅ Created `src/systems/ground-system.ts` as the single authoritative ground query (terrain + lake/island + spatial cache + platform hook).
2. ✅ Routed all main-thread ground consumers through it; workers still have their own constants (known limitation).
3. ✅ Added `CONFIG.player.eyeHeight` / `spawnEyeHeightY` and `CONFIG.ground.followLerpSpeed` / `followMaxStep`; replaced Y snaps with smooth lerp + max-step clamp.
4. ✅ Verified base-Y anchoring in `TreeBatcher` / `MushroomBatcher` via world-generation `getUnifiedGroundHeight()` delegation.
5. ✅ Added `src/debug/ground-debug.ts` with `?debugHeights=1` / `?debugPlayer=1` overlays.
6. ✅ `npm run build:ci` and `npm run test:wasm` green. Headless browser smoke tests fail due to SwiftShader WebGPU device loss (documented VM limitation).
