## 1. Tier overview

| Tier | Technology | Role today | When to use |
|------|------------|------------|-------------|
| **0** | JavaScript | Build artifacts, demo stubs, third-party glue | Almost nothing new in `src/` |
| **1** | TypeScript | Application layer — orchestration, batchers, systems | Default for all features |
| **2** | AssemblyScript → `candy_physics.wasm` | **Required** physics grid, collision, ground, particles, ECS | Hot loops > ~3 ms/frame, stateful WASM |
| **3** | C++ / Emscripten → `candy_native*.wasm` | SIMD, OpenMP, optional heavy compute | Top ~1% hotspots; always ship JS fallback |
| **4** | WebGPU compute (`src/compute/`, TSL) | GPU particles, mesh deform, noise, culling | Large parallel workloads when WebGPU available |

```
TypeScript (orchestration, batchers, UI)
        │
        ├──► AssemblyScript WASM  (required physics / grid / ground)
        ├──► Emscripten C++ WASM  (optional SIMD batches — JS fallback)
        └──► WebGPU compute       (primary particles / deform when GPU ready)
```

## 8. Stub file convention

When retiring a hand-written `.js` file:

1. Implement fully in `.ts` with types.
2. Replace `.js` with a **migration stub** that only re-exports `.ts` (see `shared-buffer-example.js`).
3. Update imports to `.ts` extension.
4. Remove stub after one release cycle if nothing external depends on the `.js` path.

**Exceptions:** `src/wasm/candy_physics.js` is a **compiler output**, not a stub.

Full policy: [`docs/archive/PERFORMANCE_MIGRATION_STRATEGY.md`](docs/archive/PERFORMANCE_MIGRATION_STRATEGY.md) § Stub Retention.

## For agents

1. Read **this file** + [`MIGRATION_TRACKER.md`](MIGRATION_TRACKER.md) before a migration pass.
2. Follow the **15% rule** — one hot function per PR when possible.
3. Always ship a **JS/TS fallback** beside new C++ or AS exports.
4. After changing C++ exports, run `pnpm run verify:emcc:manifest` (or `-- --write`) so `emscripten/exports.txt` stays in sync; CI Tier-1 enforces this.
5. Run `npm run test:wasm` after AS changes; `npm run build:ci && npm run test` for integration.
